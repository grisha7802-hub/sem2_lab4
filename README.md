# sem2_lab3

Лабораторная работа №3 (2 семестр)  
Вариант: **№3 — Стек (Stack)**

## Что реализовано

### Базовые АТД (наследие ЛР2)
- `DynamicArray<T>`
- `LinkedList<T>`
- `Sequence<T>`
- `ArraySequence<T>`
- `ListSequence<T>`

### Целевой АТД
- `Stack<T>` на базе `ArraySequence<T>`

### Базовые операции стека
- `Push(const T&)`
- `Pop()`
- `Peek()`
- `IsEmpty()`
- `GetSize()`

### Дополнительные операции по ТЗ
- `Map`
- `Where`
- `Reduce`
- `Concat` (конкатенация)
- `GetSubStack(start, end)` (извлечение подпоследовательности)
- `FindSubStack(pattern)` (поиск вхождения подпоследовательности)

### Безопасные операции (try-семантика)
- `TryPop() -> Option<T>`
- `TryPeek() -> Option<T>`

## ЛР4 (добавлено)

На базе ЛР3 добавлены:
- `LazySequence<T>` с мемоизацией материализованных элементов
- `SequenceLength` (конечная/бесконечная мощность)
- `Generator<T>` для последовательного порождения элементов
- семейство потоков:
  - `Stream<T>` (`Map`, `Where`, `Reduce`, `Take`, `Zip`)
  - `ReadOnlyStream<T>` (`Read`, `Seek`, `IsEndOfStream`, `Open`, `Close`)
  - `WriteOnlyStream<T>` (`Write`, `Open`, `Close`)
- чтение потоков из строки и файла, запись в `Sequence` и файл
- `OnlineStatistics<T>` как тематическая задача обработки потока событий
- `Either<L, R>` для результата, который хранит одно из двух возможных значений

Выбран тематический вариант: сбор статистики потока событий в режиме онлайн.
Демонстрационные данные включают бесконечный поток натуральных чисел, поток
Фибоначчи и ручной ввод числового потока.

Новые цели:
- `lr4_console` — консольное ручное тестирование ЛР4
- `lr4_tests` — модульные тесты ЛР4
- `lr4_gtk` — графический интерфейс ЛР4
- `lr4_bench` — стресс-проверки и замеры на больших потоках

Проверки ЛР4 покрывают:
- конечные и бесконечные `LazySequence`
- `Map`, `Where`, `Reduce`, `Concat`, `Zip`, `RemoveAt`
- `Generator`
- чтение/запись потоков в память и файл
- ручной поток из строки
- онлайн-статистику (`count`, `sum`, `min`, `max`, `mean`, `median`, оконные `min/max/mean`)
- `Either<L, R>`: левое/правое значение, безопасные fallbacks, ошибка при доступе не к той стороне
- граничный прогон на 100000 элементов ленивого потока
- файловый поток на 50000 элементов в тестах и 200000 элементов в benchmark

### Краткая теория

`LazySequence<T>` хранит уже вычисленный префикс в `ArraySequence<T>` и правило
порождения следующего элемента. При первом обращении к индексу последовательность
достраивает все недостающие элементы до этого индекса. Повторное обращение к уже
материализованному элементу идет напрямую в массив, поэтому при работе с одним и
тем же префиксом средняя стоимость доступа стремится к `O(1)`.

Потоки отличаются от ленивых последовательностей тем, что чтение продвигает
позицию потока вперед. Для потоков реализованы отдельные интерфейсы чтения и
записи, а также смешанный поток в памяти. Тематическая задача использует поток
как источник событий и обновляет статистику за один проход без хранения внешних
промежуточных коллекций.
